Objetivo da Página

O objetivo dessa página é documentar como são armazenados os personagens Miis*, criados pela Nintendo inicialmente para o console Nintendo Wii porém usado até os dias de hoje.[I]

* Mii é um avatar personalizável utilizado em alguns consoles de videogames da Nintendo e aplicativos para smartphones.

Distribuição de Arquivos

Os arquivos de Miis, até o momento, possuem 7 diferentes formatos, cada um com suas peculiaridades e diferentes distribuições de bits.[II]

Até o momento, são esses os formatos, separados por plataforma:

Nintendo Wii

RCD

É o padrão em jogos de Nintendo Wii, possui toda a informação necessária para caracterizar um Mii.[II]

RSD

Comparado ao RCD, é extremamente similar porém com uma única exceção: possui 2 bytes ao final do arquivo que servem como checksum.[II]

Nintendo Wii U / Nintendo 3DS

CFSD

É o padrão em jogos de Nintendo Wii U e Nintendo 3DS, possui toda a informação necessária para caracterizar um Mii nessas plataformas. Um fato importante é que, nessa extensão, os arquivos possuem o valor 308 no quarto byte, indicando que os Miis nessa extensão foram criados no 3DS.[II]

FFSD

Comparado ao CFSD, é extremamente similar porém com uma única exceção: ao invés de conter o valor 308 no quarto byte, possui o valor 408. Esse valor representa que o Mii em questão foi criado no Wii U.[II]

Nintendo Switch

NFSD

É o formato encontrado na NAND do sistema Nintendo Switch. Sabe-se, no entanto, que os jogos possuem outra extensão predominante: o CHARINFO. Além disso, sabe-se que esse arquivo possui bytes dedicados ao checksum.[II]

CHARINFO

É o padrão em jogos de Nintendo Switch, possui toda a informação necessária para caracterizar um Mii no Switch. Diferentemente do NFSD, essa extensão não possui checksum.[II]

Site Oficial da Nintendo

MNMS

Esse formato é somente visto no Criador de Miis no site oficial da Nintendo como um cache local.

Tecnicamente é um arquivo-texto ao invés de um binário.[II]
